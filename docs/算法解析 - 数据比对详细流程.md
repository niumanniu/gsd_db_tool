# 数据比对算法详解

## 前置条件

- **批次大小**：batchSize = 1000
- **主键类型**：int64
- **读取方式**：`WHERE pk >= ? ORDER BY pk LIMIT batchSize+1`
- **nextKey 选择**：较小的值（修复后）

---

## 核心算法流程

```
                    ┌─────────────────┐
                    │   开始比对      │
                    └────────┬────────┘
                             │
                    ┌────────▼────────┐
                    │ 初始化：        │
                    │ sourceRemaining │
                    │ targetRemaining │
                    └────────┬────────┘
                             │
              ┌──────────────┴──────────────┐
              │        主循环开始           │
              └──────────────┬──────────────┘
                             │
              ┌──────────────▼──────────────┐
              │ 1. 读取批次数据             │
              │ sourceBatch, sourceNext     │
              │ targetBatch, targetNext     │
              └──────────────┬──────────────┘
                             │
              ┌──────────────▼──────────────┐
              │ 2. 合并到 remaining map     │
              │ sourceMap = sourceRemaining │
              │ targetMap = targetRemaining │
              │ + 当前批次数据              │
              └──────────────┬──────────────┘
                             │
              ┌──────────────▼──────────────┐
              │ 3. 遍历 sourceMap 比对      │
              │   for key, srcRow := range  │
              │     if target 有相同 key:   │
              │       if Hash 相同：        │
              │         ✓ delete 跳过       │
              │       else:                 │
              │         ~ 详细比对          │
              │         → diff.Modified     │
              │         delete              │
              │     else:                   │
              │       保留在 map 中         │
              └──────────────┬──────────────┘
                             │
              ┌──────────────▼──────────────┐
              │ 4. 计算 nextKey = min()     │
              │    (选较小值确保不遗漏)     │
              └──────────────┬──────────────┘
                             │
                    ┌────────▼────────┐
                    │ 还有更多数据？  │
                    └───┬────────┬────┘
                      是 │        │ 否
                        │        │
              ┌─────────┘        └──────────┐
              │                             │
              │    ┌───────────────────────┐
              │    │ 5. 处理剩余记录       │
              │    │ sourceRemaining       │
              │    │   → diff.Removed (-)  │
              │    │ targetRemaining       │
              │    │   → diff.Added (+)    │
              │    └───────────────────────┘
              │
              │
     ┌────────▼────────┐
     │ currentKey =    │
     │ nextKey         │
     └────────┬────────┘
              │
              └──────┐
                     │
              ┌──────┴──────┐
              │  下一批次   │
              └─────────────┘
```

## 不一致处理详细流程

### 场景 A：主键不匹配（新增/删除）

**数据分布**：
```
源表第 2 批：{3, 5}     ← 缺 4
目标表第 2 批：{3, 4}   ← 缺 5
```

**处理流程**：
```
┌─────────────────────────────────────────────────────────────┐
│ 第 1 步：合并到 remaining map                                │
├─────────────────────────────────────────────────────────────┤
│ sourceMap = {} ∪ {3, 5} = {"3": row3, "5": row5}            │
│ targetMap = {} ∪ {3, 4} = {"3": row3, "4": row4}            │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│ 第 2 步：遍历 sourceMap 比对                                 │
├─────────────────────────────────────────────────────────────┤
│ key="3": target 有 → 比对相同 → delete("3") ✓               │
│ key="5": target 没有 → 保留在 sourceMap                     │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│ 第 3 步：比对后状态                                          │
├─────────────────────────────────────────────────────────────┤
│ sourceMap = {"5": row5}  ← 未匹配，保留到下一批             │
│ targetMap = {"4": row4}  ← 未匹配，保留到下一批             │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│ 第 4 步：计算 nextKey                                        │
├─────────────────────────────────────────────────────────────┤
│ sourceNext = 5                                              │
│ targetNext = 5                                              │
│ nextKey = min(5, 5) = 5                                     │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│ 第 5 步：进入下一批 (currentKey = 5)                         │
├─────────────────────────────────────────────────────────────┤
│ sourceRemaining = {"5": row5}  ← 继续参与比对               │
│ targetRemaining = {"4": row4}  ← 继续参与比对               │
└─────────────────────────────────────────────────────────────┘
```

### 场景 B：数据修改（Hash 不同）

**数据分布**：
```
源表第 2 批：{3, 4}   name: "Alice"
目标表第 2 批：{3, 4}   name: "Bob"  ← 修改了
HashFilter = true
```

**处理流程**：
```
┌─────────────────────────────────────────────────────────────┐
│ 第 1 步：读取批次（带 Hash）                                 │
├─────────────────────────────────────────────────────────────┤
│ sourceBatch = [                                             │
│   {id:3, name:"Alice", __row_hash__: 12345},                │
│   {id:4, name:"Charlie", __row_hash__: 67890}               │
│ ]                                                           │
│ targetBatch = [                                             │
│   {id:3, name:"Bob", __row_hash__: 54321},   ← Hash 不同    │
│   {id:4, name:"Charlie", __row_hash__: 67890}               │
│ ]                                                           │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│ 第 2 步：检查批次 Hash 是否跳过                              │
├─────────────────────────────────────────────────────────────┤
│ sourceBatchHash = "abc123..."                               │
│ targetBatchHash = "def456..."  ← 不同，不跳过               │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│ 第 3 步：构建 map                                            │
├─────────────────────────────────────────────────────────────┤
│ sourceMap = {"3": row3_src, "4": row4_src}                  │
│ targetMap = {"3": row3_tgt, "4": row4_tgt}                  │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│ 第 4 步：逐行比对                                            │
├─────────────────────────────────────────────────────────────┤
│ key="3":                                                    │
│   srcHash = 12345, tgtHash = 54321  ← 不同                  │
│   → compareRow() → ["name: Alice -> Bob"]                   │
│   → diff.Modified.append({Key:"3", ...})                    │
│   → delete("3")                                             │
│                                                             │
│ key="4":                                                    │
│   srcHash = 67890, tgtHash = 67890  ← 相同                  │
│   → 跳过详细比对                                            │
│   → delete("4")                                             │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│ 第 5 步：比对结束                                            │
├─────────────────────────────────────────────────────────────┤
│ sourceMap = {}                                              │
│ targetMap = {}                                              │
│ diff.Modified = [{Key:"3", Source:{name:"Alice"},           │
│                   Target:{name:"Bob"},                      │
│                   Changes:["name: Alice -> Bob"]}]          │
└─────────────────────────────────────────────────────────────┘
```

### 场景 C：批次大小不同（最复杂）

**数据分布**：
```
源表第 1 批：{1, 2, 3}     ← 1000 行
目标表第 1 批：{1, 2, 4}   ← 1000 行（缺 3，多 4）
```

**处理流程**：
```
┌─────────────────────────────────────────────────────────────┐
│ 第 1 步：读取批次                                            │
├─────────────────────────────────────────────────────────────┤
│ sourceBatch = {1, 2, 3}, sourceNext = 1001                  │
│ targetBatch = {1, 2, 4}, targetNext = 1002  ← 多读一个      │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│ 第 2 步：构建 map                                            │
├─────────────────────────────────────────────────────────────┤
│ sourceMap = {"1": r1, "2": r2, "3": r3}                     │
│ targetMap = {"1": r1, "2": r2, "4": r4}                     │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│ 第 3 步：比对                                                │
├─────────────────────────────────────────────────────────────┤
│ "1" → 匹配 → delete                                         │
│ "2" → 匹配 → delete                                         │
│ "3" → target 没有 → 保留在 sourceMap                        │
│ "4" → source 没有 → 保留在 targetMap                        │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│ 第 4 步：计算 nextKey                                        │
├─────────────────────────────────────────────────────────────┤
│ nextKey = min(1001, 1002) = 1001                            │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│ 第 5 步：进入第 2 批                                         │
├─────────────────────────────────────────────────────────────┤
│ sourceRemaining = {"3": r3}  ← 保留                         │
│ targetRemaining = {"4": r4}  ← 保留                         │
│                                                             │
│ 第 2 批读取：                                               │
│ sourceBatch = {1001, 1002, ...}                             │
│ targetBatch = {1001, 1002, ...}                             │
│                                                             │
│ 合并：                                                      │
│ sourceMap = {"3": r3, "1001": r1001, ...}                   │
│ targetMap = {"4": r4, "1001": r1001, ...}                   │
└─────────────────────────────────────────────────────────────┘
```

## 不一致类型与处理策略总结

| 不一致类型 | 触发条件 | 处理方式 | 最终结果 |
|------------|----------|----------|----------|
| **主键不匹配** | 某行在一方不存在 | 保留在 remaining，继续后续批次 | 循环结束后确认 → Added/Removed |
| **数据修改** | Hash 预筛选失败 | 详细比对 → 记录差异 | 立即记录到 diff.Modified |
| **批次大小不同** | 数据分布不同 | 保留在 remaining，继续后续批次 | 可能重复读取，但保证准确性 |

## 缺失数据对批次的影响

### 问题：target 缺失 2 条数据，总批次增加多少？

**数据分布**：
```
源表：1-10000（共 10000 行）
目标表：缺 500 和 600（共 9998 行）
batchSize = 1000
```

**第 1 批分析**：
```
┌─────────────────────────────────────────────────────────────┐
│ source: SELECT * WHERE pk >= 1 LIMIT 1001                   │
│ → 返回 1-1000 (1000 行)，第 1001 行 = 1001                   │
│ → sourceNext = 1001                                         │
├─────────────────────────────────────────────────────────────┤
│ target: SELECT * WHERE pk >= 1 LIMIT 1001                   │
│ → 跳过 500,600 取 1001 行                                   │
│   1-499   → 499 行                                          │
│   501-599 → 99 行                                           │
│   601-1000 → 400 行                                         │
│   1001 → 1 行                                               │
│   1002 → 1 行                                               │
│   1003 → 1 行 (第 1001 行)                                  │
│ → targetNext = 1003                                         │
├─────────────────────────────────────────────────────────────┤
│ nextKey = min(1001, 1003) = 1001                            │
└─────────────────────────────────────────────────────────────┘
```

**结论**：
| 指标 | 结果 |
|------|------|
| **总批次** | 10 次 |
| **比正常增加** | +0 次 |
| **原因** | 缺失数据在第 1 批内部被"消化"，后续批次对齐 |
